Refine the case study landing/hero section responsiveness.

Problem:
On a 15.6-inch laptop screen, the Safeway case study hero is too large vertically. The heading, top spacing, and metadata cards take up too much space, so the content feels zoomed-in and oversized. On a larger monitor or zoomed-out view, it looks much more balanced.

Goal:
Make the case study hero adapt better to laptop screens like 1366x768, 1440x900, and 1536x864. The laptop version should look closer to the balanced second screenshot, where the title, subtitle, and metadata cards are visible more comfortably.

Important:
- Do not redesign the whole case study page.
- Do not change the warm editorial color palette.
- Do not change the text content.
- Do not change the section order.
- Only improve hero sizing, spacing, and responsive behavior.

Tasks:

1. Reduce excessive top spacing
The top area between the navbar and the “Back to Portfolio” link is too tall on laptop screens.
Reduce the vertical spacing on medium/laptop viewport heights.

Use viewport-aware CSS such as:
@media (max-height: 850px)
@media (max-width: 1536px)
@media (max-width: 1440px)

2. Scale the case study title more intelligently
The title “Safeway Grocery Experience Redesign” is too large on laptop screens.

Use clamp() for the case study title font size.
The title should remain bold and impressive on large screens, but slightly smaller on laptop screens.

Suggested direction:
- large desktop: keep strong oversized heading
- laptop: reduce title size enough so it does not dominate the whole first viewport
- mobile: stack naturally and stay readable

3. Tighten title line-height and spacing
Adjust:
- line-height of the case study title
- margin below the title
- spacing between label, title, subtitle, and metadata grid

The title should still feel premium, but not oversized.

4. Improve metadata card layout
The metadata cards should appear earlier and more compactly on laptop screens.

Adjust:
- card padding
- grid gap
- vertical spacing above metadata grid
- card height if needed

Do not make them cramped. Just reduce unnecessary empty space.

5. Reduce case study hero section height
The case study hero should not require excessive scrolling before the user sees the metadata cards.

On laptop screens, the following should be comfortably visible within the first viewport:
- navbar
- Back to Portfolio link
- UX Case Study label
- title
- subtitle
- at least the first few metadata cards

6. Keep large desktop appearance polished
Do not make the large monitor version worse.
Preserve the elegant large-screen style, but make the laptop version more compact.

7. Use clean CSS
Update only the CSS related to the case study hero/intro section.

Possible classes to adjust:
- .case-hero
- .case-hero-inner
- .case-eyebrow
- .case-title
- .case-subtitle
- .case-meta-grid
- .case-meta-card
- .back-link

8. Add height-based responsiveness
This issue is caused by available vertical screen space, not only screen width.
Add media queries for shorter viewports.

Example direction:
When viewport height is below 850px:
- reduce hero top padding
- reduce title font size slightly
- reduce title margin
- reduce metadata card padding
- reduce gaps

When viewport height is below 760px:
- compact the hero further
- ensure it still looks intentional, not squeezed

9. Preserve desktop and mobile behavior
After updating, check:
- 1920x1080
- 1536x864
- 1440x900
- 1366x768
- tablet
- mobile

Expected result:
The case study landing should look like the second screenshot on laptop screens: balanced, compact, polished, and easier to scan without feeling zoomed-in.

Do not modify other sections of the case study page.


Suggested CSS behavior:
- case study title: font-size: clamp(4rem, 7vw, 8.5rem);
- for max-height: 850px, reduce title to clamp(3.5rem, 6vw, 6.5rem)
- for max-height: 760px, reduce title to clamp(3rem, 5.5vw, 5.5rem)
- reduce case hero padding-top and padding-bottom on shorter screens
- reduce metadata card padding from large desktop values to around 1.5rem on laptop screens
- keep the page max-width centered and consistent with the rest of the site


Apply the same responsive philosophy used for the homepage hero to the case study hero, but tune it for the case study layout.
